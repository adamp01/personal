I was recently optimising a few processes when I ended up performing this comparison, which I found to be quite interesting. 
It is by no means a complete experiment and there are plenty of ways this could be further expanded. I had a byte array, 
and in that byte array was a known tag with a certain number of null bytes following it. I needed to be able to identify 
this tag in the byte array to discern the index at which it is safe to insert additional bytes of data, as only null bytes will 
be replaced. As the tag is 14 bytes long I summised that an unrolled for loop would be the quickest comparison method but 
wanted to verify this against other methods that I could think of. The string representation of the tag in question is 
<CodeInline>\_\_MOZCUSTOM\_\_</CodeInline> and this is almost certainly going to be found within the first 10kB of the array.<br/>
We now have enough information to set up a test:

- Generate an unsigned, 8-bit integer array with 10,000 values (close enough).
- Populate this with random values.
- Replace the last 14 values with the tag bytes.
